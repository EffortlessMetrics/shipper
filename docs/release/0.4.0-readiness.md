## Preflight

Local preflight was run from a copied `shipper` binary with an isolated
`CARGO_TARGET_DIR`. Running preflight through `cargo run` on Windows locks
`target/debug/shipper.exe` and makes the dry-run verifier fail with
`Access is denied`.

Command:

```bash
shipper preflight --preflight-only --allow-dirty --skip-ownership-check --state-dir target/shipper-preflight-proof --format json
```

Result:

| Field | Value |
|---|---|
| `plan_id` | `5d63c5b0725a59a01c1fa1406220808f5a7b1a166c0ddf76d3ba97d13e6feeb5` |
| `finishability` | `not_proven` |
| `token_detected` | `true` |
| `ownership_verified` | `false` for all packages, because `--skip-ownership-check` was used |
| `dry_run_passed` | `true` for all 13 packages |
| state sidecar | `target/shipper-preflight-proof/preflight_workspace_verify.txt` |

The `not_proven` verdict is expected for this local proof because ownership was
skipped and the branch had uncommitted proof changes. PR CI must provide the
strict clean-tree signal after the artifact is committed.
